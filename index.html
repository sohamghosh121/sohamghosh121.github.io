Selected Projects

Memory Augmented Hierarchical RL

A significant problem in RL is memory - remembering what an agent saw long back and using that information. In LSTM based models, spatial information (such as layouts, maze specific information) is often lost. We first created a customizable, OpenAI Gym form of the Berkeley Pacman environment (here) and then we built on top of two recent papers in hierarchical and memory-augmented RL - MLSH and Neural Map to try to get Pacman to do better in partially observable environments.
Technologies used : TensorFlow, Pytorch
Worked with : Sai Bhargav Yalamanchi, Varsha Chinnaobireddy (CMU)

Natural Parameter Recurrent Networks

Bayesian Neural Networks are cool - because they give uncertainty estimates about their predictions (and I got really fascinated with them) but are usually trained using sampling based methods/variational inference because the posterior is intractable to compute. But, this paper found a really smart way of doing this by approximate moment matching and also saying each neuron is a distribution (instead of single value). We extended this approach to Recurrent Neural Networks (we implemented them for GRUs), and voila - you can now have uncertainty estimates for GRU-based sentiment analysis models.
Technologies used : Pytorch
Worked with : Sumedha Singla (UPitt)

RHS: A regular expression DSL in Javascript

My team and I realised that composing complex regular expressions are difficult, confusing, and can often be wrong, and are definitely hard to read. What code is really easy to read? The jQuery syntax of chaining method calls. Hence we wanted to create a JavaScript DSL that would allow users to compose regular expressions (and reuse them) in jQuery like syntax. This DSL also allowed for auto generation of a NFA equivalent to the regular expression.
Technologies used : JavaScript, Graphviz

Block-based Image Compression in OpenCV

I implemented a distributed image compression algorithm for a real time imaging application based on the idea of compressive sensing. In this, different blocks were encoded at different measurement rates. The Scrambled Block Hadamard Ensemble (SBHE) matrix was used as the measurement matrix, Daubechies 9/7 wavelets as the sparsifying transform, and the SpaRSA algorithm for reconstruction. The aim of this project was to investigate the tradeoff between reconstruction speed, PSNR and impact on pedestrian detection accuracy on street images.
Technologies used : C++, OpenCV
Worked with : Alexander Hanel, Dr. Felix Klanner (Technical University of Munich)
